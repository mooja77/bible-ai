# Security Policy

## Reporting

Please report security issues privately by opening a GitHub security advisory when available, or by contacting the repository owner directly.

Do not open a public issue for vulnerabilities involving credential storage, provider keys, local data exposure, or export leaks.

## Scope

Security-sensitive areas include:

- Provider credential storage and migration.
- Backup/export redaction.
- Council source drawers and release evidence.
- Managed gateway token handling.
- Local SQLite user data.
- Installer packaging and update flows.

## Supported Versions

This project is early-stage. Security fixes target the main branch until formal release branches exist.
